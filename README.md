# Renuga Sree | AI/ML Engineer Portfolio

A production-grade, recruiter-winning technical product showcase. Designed for high-impact visibility in Data Science and AI/ML Engineering roles.

## 🎯 The Objective
This portfolio is engineered to demonstrate **production exposure**. It goes beyond model training to showcase end-to-end system thinking, API design, and scalable deployment practices.

## 🚀 Technical Architecture

- **Core:** [Next.js 15](https://nextjs.org/) (App Router)
- **Engine:** [TypeScript](https://www.typescriptlang.org/) for type-safe system logic
- **Styling:** [Tailwind CSS](https://tailwindcss.com/) with a custom Glassmorphism design system
- **Interactions:** [Framer Motion](https://www.framer.com/motion/) for premium micro-animations
- **Infrastructure:** [GitHub Actions](https://github.com/features/actions) for automated CI/CD to GitHub Pages

## 🛠️ Key Product Features

- **Adaptive Case Study:** A deep dive into the *Adaptive AI Learning Assistant*, featuring architecture breakdowns, mastery logic code snippets, and production metrics.
- **Engineering Showcase:** Project cards structured with *Problem → Solution → Tech → Outcome* to simulate real-world product reporting.
- **Impact-Driven Journey:** Professional history rewritten to highlight business outcomes and technical scalability rather than just tasks.
- **Unified Contact Interface:** A high-impact "Mega Footer" that combines brand identity with conversion-focused CTAs.

## 🏃 Engineering Setup

### 1. Installation
```bash
npm install
```

### 2. Local Development
```bash
npm run dev
```

### 3. Production Build (Static Export)
```bash
npm run build
```
The static site will be generated in the `/out` directory.

## 📦 Deployment Strategy (GitHub Pages)

This project is configured for automated deployment via GitHub Actions.

1. **Repository Settings:** Ensure your GitHub repository name is `Portfolio`.
2. **Enable Pages:** Go to `Settings > Pages` and set the Source to **GitHub Actions**.
3. **Automated Flow:** Any push to the `main` branch triggers the `.github/workflows/deploy.yml` pipeline, which handles the build and deployment automatically.
